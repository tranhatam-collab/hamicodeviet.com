## 1. Nguyên tắc triển khai

- Mỗi phase có scope, dependencies, entry gate, exit gate, evidence và rollback.
- Không build marketplace trước identity, trust, payment và legal.
- Không build AI mở trước age/consent/policy.
- Không build animation trước core learning flow và security.

## 2. Roadmap 12 tháng

### Phase 0 - Foundation (0-30 ngày)

- Repository, baseline audit, Founder Lock.
- Domain/DNS/SSL, environments, CI/CD.
- Architecture, data model, design system.
- Auth foundation, guardian/consent model.
- Privacy/AI/child safety draft.
- Monitoring, backup, secret hygiene.

**Exit:** staging xanh, no secret leak, domain resolved, architecture approved.

### Phase 1 - Public & Learning MVP (31-90 ngày)

- Public website song ngữ.
- 5 paths, 30 lessons, 100 demos.
- Placement assessment, progress, projects.
- Parent and teacher landing/resources.
- SEO, analytics, accessibility review.

**Exit:** 500 verified accounts và 200 completed projects là KPI pilot, không phải release requirement.

### Phase 2 - Account, Guardian & Paid Academy (Tháng 4-6)

- Age-aware signup, guardian consent.
- Family/Creator plans, checkout, webhook, entitlement.
- 3 cohort đầu tiên.
- Parent reports, assessments, certificate baseline.

### Phase 3 - AI Agent MVP (Tháng 4-7, song song có kiểm soát)

- AI gateway, Tutor, Code Coach, Assessment Draft.
- Quota, audit, cost dashboard, kill switch.
- Eval datasets và safety tests.

### Phase 4 - CodeLab (Tháng 6-9)

- HTML/CSS/JS/Python editor và sandbox.
- Versioning, tests, submissions, review.
- Abuse and security testing.

### Phase 5 - Portfolio & Marketplace Pilot (Tháng 8-11)

- Evidence portfolio, privacy modes, certificate verify.
- Creator onboarding, license, product review, test sales.
- Guardian payout design; marketplace feature-flagged.

### Phase 6 - School Pilot & Globalization (Tháng 10-12)

- Classroom, assignments, reporting, school license.
- International pricing/landing và jurisdiction controls.
- 3-10 pilots, không mở đại trà nếu data isolation chưa pass.

## 4. Definition of Done

Một task chỉ Done khi:

- Acceptance criteria đạt.
- Code/content review xong.
- Test liên quan pass.
- Docs cập nhật.
- Security/privacy impact xem xét.
- Preview verified.
- Evidence gắn vào issue/PR.
- Nếu public: production verified.
